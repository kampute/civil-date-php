# کتابخانهٔ تاریخ مدنی برای پی‌اچ‌پی

فارسی | [English](./README.md)

این کتابخانه تاریخ‌های جلالی، میلادی و اسلامی را به‌صورت یکپارچه مدل‌سازی می‌کند. در تقویم جلالی، نوروز و سال کبیسه از قاعده رسمی تقویم ایران و زمان اعتدال بهاری به دست می‌آیند، نه از چرخه‌های تقریبی و تکرارشونده.

## هدف

بسیاری از کتابخانه‌های تاریخ جلالی برای تشخیص سال کبیسه از الگوهای عددی ثابت استفاده می‌کنند. این روش‌ها ساده و سریع‌اند، اما در بعضی سال‌ها ممکن است نوروز را یک روز جلوتر یا عقب‌تر از تقویم رسمی قرار دهند.

در این کتابخانه، آغاز هر سال جلالی از روی زمان اعتدال بهاری در وقت رسمی ایران تعیین می‌شود. اگر اعتدال پیش از ظهر رخ دهد، همان روز نوروز است؛ اگر در ظهر یا بعد از آن رخ دهد، روز بعد نوروز خواهد بود. طول سال و کبیسه بودن آن هم از فاصله دو نوروز پیاپی به دست می‌آید.

## قابلیت‌ها

- پشتیبانی جامع از تاریخ در تقویم‌های جلالی، میلادی و اسلامی.
- ساخت تاریخ از منابع گوناگون: تبدیل از تقویم دیگر، از روز سال، از چندمین رخ‌داد یک روز هفته در ماه یا سال، و مانند آن.
- قالب‌بندی و خواندن تاریخ از متن، شامل نام ماه‌ها، روزها، فصل‌ها و دوره‌ها و اعداد به‌صورت رقمی یا حروفی، متناسب با زبان.
- مقایسهٔ تاریخ‌ها بر پایه روز، هفته، ماه، فصل و سال.
- ایجاد تاریخ جدید با افزودن یا کاهش روز، ماه یا سال.
- تشخیص سال کبیسه و طول سال جلالی بر پایه نوروز نجومی.
- تعیین طول معتبر ماه‌های اسلامی، برای برنامه‌هایی که به داده‌های رسمی طول ۲۹ یا ۳۰ روزهٔ ماه‌ها نیاز دارند.
- پشتیبانی از نسخهٔ ⁧۸٫۱⁩ و بالاتر پی‌اچ‌پی، بارگذاری خودکار بر پایهٔ استاندارد پی‌اس‌آر-۴ و بدون وابستگی اجرایی.

## نصب

نصب کتابخانه:

```bash
composer require kampute/civil-date
```

## شروع سریع

```php
<?php

require_once __DIR__ . '/vendor/autoload.php';

use Kampute\CivilDate\Calendar;
use Kampute\CivilDate\Calendars\JalaliCalendar;
use Kampute\CivilDate\DayOfWeek;
use Kampute\CivilDate\JalaliDate;

$nowruz = JalaliDate::fromGregorianDate(2025, 3, 21);

echo $nowruz;
// 1404/01/01

echo $nowruz->toIso8601DateString();
// 2025-03-21

echo $nowruz->toCalendar(Calendar::Gregorian);
// 2025/03/21

echo $nowruz->isLeapYear ? 'leap' : 'common';
// common

$equinox = JalaliCalendar::instance()->vernalEquinox(1404);

echo $equinox->format('Y-m-d H:i:s P');
// 2025-03-20 12:31:36 +03:30

$firstSaturday = JalaliDate::fromNthDayOfWeekInMonth(
    1404,
    1,
    1,
    DayOfWeek::Saturday
);

echo $firstSaturday;
// 1404/01/02
```

## قالب‌بندی و خواندن تاریخ

الگوهای قالب‌بندی به الگوهای رایج تاریخ در پی‌اچ‌پی نزدیک‌اند و می‌توانند چند تقویم را در یک خروجی نشان دهند. زبان و منطقه پیش‌فرض فارسی است؛ بنابراین خروجی جلالی بدون تنظیم اضافه با ارقام و نام‌های فارسی تولید می‌شود.

```php
use Kampute\CivilDate\JalaliDate;

$date = new JalaliDate(1405, 2, 14);

echo $date->format('l j F Y');
// دوشنبه ۱۴ اردیبهشت ۱۴۰۵

echo $date->format('Y/m/d [gregorian:Y-m-d] [islamic:Y/m/d]');
// ۱۴۰۵/۰۲/۱۴ ۲۰۲۶-۰۵-۰۴ ۱۴۴۷/۱۱/۱۷

echo $date->format('روز R و هفته K سال Y');
// روز چهل و پنجم و هفته هفتم سال ۱۴۰۵

$parsed = JalaliDate::parse('دوشنبه ۱۴ اردیبهشت ۱۴۰۵', 'l j F Y');

echo $parsed;
// 1405/02/14

$lastWednesday = JalaliDate::parse('آخرین چهارشنبه ۱۴۰۵', 'K l Y');

echo $lastWednesday;
// 1405/12/26

$ordinalDate = JalaliDate::parse('⁧روز دویست و پنجاه و ششم سال ۱۴۰۶⁩', '⁧روز R سال Y⁩');

echo $ordinalDate;
// 1406/09/10
```

## محلی‌سازی

تعریف‌های زبان و منطقه از منطق تقویم جدا هستند. فارسی به‌صورت پیش‌فرض فعال است، انگلیسی و فارسی افغانستان هم آماده‌اند، و اگر پروژه شما واژگان یا زبان دیگری لازم داشته باشد می‌توانید تعریف خودتان را اضافه کنید.

```php
use Kampute\CivilDate\JalaliDate;

$date = new JalaliDate(1405, 2, 14);

echo $date->format('l j F Y', ['locale' => 'en']);
// Monday 14 Ordibehesht 1405

echo $date->format('l j F Y', ['locale' => 'fa-AF']);
// دوشنبه ۱۴ ثور ۱۴۰۵

$parsed = JalaliDate::parse('Monday 14 Ordibehesht 1405', 'l j F Y', ['locale' => 'en']);

echo $parsed;
// 1405/02/14
```

## طول ماه‌های اسلامی

تقویم اسلامی به‌صورت پیش‌فرض از نسخهٔ مدنی و جدولی استفاده می‌کند. اگر برنامهٔ شما به طول رسمی یا اعلام‌شدهٔ ماه‌ها دسترسی دارد، آن را هنگام راه‌اندازی برنامه و پیش از اولین استفاده از تقویم اسلامی ثبت کنید.

```php
use Kampute\CivilDate\Calendars\IslamicCalendar;

IslamicCalendar::instance()->setAuthoritativeMonthLengths([
    1446 => [
        9 => 29,
        10 => 30,
    ],
]);
```

برای ماه‌هایی که مقداری ثبت نشده باشد، طول جدولی استفاده می‌شود. چون طول یک ماه می‌تواند مرز تاریخ‌های بعدی را تغییر دهد، این داده‌ها باید فقط یک بار و در شروع برنامه بارگذاری شوند.

## دقت و بازه پشتیبانی

تبدیل‌های جلالی بر پایه تخمین نجومی اعتدال بهاری در وقت رسمی ایران انجام می‌شوند. این روش برای بازهٔ تاریخی گسترده‌ای طراحی شده و در اجراهای مختلف، نتیجه‌ای قطعی و تکرارپذیر می‌دهد.

تنها سال‌هایی حساس‌اند که لحظهٔ اعتدال بهاری آن‌ها بسیار نزدیک به ظهر وقت ایران باشد. طبق قاعدهٔ رسمی، اگر این لحظه پیش از ظهر باشد، نوروز همان روز است؛ و اگر از ظهر به بعد باشد، نوروز روز بعد خواهد بود. در نتیجه، اگر دو محاسبهٔ نجومی، لحظهٔ اعتدال را در دو طرف این مرز قرار دهند، ممکن است نتیجهٔ نهایی متفاوت شود.

بازهٔ پشتیبانی‌شده برای تاریخ جلالی، از ۱۶۲۲ سال پیش از مبدأ تا ۲۳۷۸ سال پس از آن است که معادل ۱۰۰۱ پیش از میلاد تا ۳۰۰۰ میلادی است. توجه داشته باشید که سال صفر تعریف نشده است.

## توسعه

نصب وابستگی‌ها:

```bash
composer install
```

اجرای تست‌ها:

```bash
composer test
```

اجرای تحلیل ایستا و بررسی سبک کد:

```bash
composer analyse
composer run cs-check
composer run cs-fix
```

ساخت سایت مستندات در محیط محلی:

```bash
composer docs
```

بعد از ساخت، فایل ⁦`.site/index.html`⁩ را باز کنید.
